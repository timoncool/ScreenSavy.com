# 🚀 ScreenSavy.com - Архитектурный Рефакторинг

## 📋 Обзор

Полная оптимизация и реструктуризация проекта для:
- ⚡ Максимальная производительность
- 🎨 Сохранение всех визуальных эффектов
- 🔧 Подготовка к новым фичам (видео плеер, пользовательские скринсейверы)
- 📦 Улучшенная модульность и переиспользование кода

## 🎯 Критические проблемы (найденные)

### 1. Производительность смены цветов
**Проблема:** 510+ перерисовок на каждый переход цвета
- Множественные `setRgb()` и `setCurrentHex()` вызовы
- CSS transition конфликтует с частыми обновлениями
- Использование setTimeout вместо requestAnimationFrame

**Решение:**
- CSS переменные для цвета фона
- requestAnimationFrame для анимации
- Батчинг обновлений состояния
- Отдельный слой для анимации

### 2. Монолитный компонент
**Проблема:** MainExperience.tsx - 1934 строки
- Сложная поддержка
- Невозможность переиспользования
- Все в одном файле

**Решение:**
- Модульная архитектура
- Разделение на логические компоненты
- Переиспользуемые хуки

### 3. Отсутствие глобального состояния
**Проблема:** Props drilling через множество уровней
**Решение:** Context API для общих данных

## 🔄 ФАЗА 1: Критические оптимизации (День 1)

- Убрать inline styles где возможно
- Оптимизировать transitions

### 1.2 Хук useColorAnimation
**Файл:** `hooks/useColorAnimation.ts`
- Использовать requestAnimationFrame
- Батчинг обновлений
- Оптимизированная интерполяция цветов
- Поддержка pause/resume

### 1.3 Оптимизация рендеринга
**Файлы:** Все компоненты UI
- React.memo для всех тяжелых компонентов
- useMemo для вычисляемых значений

## 🏗️ ФАЗА 2: Модульная архитектура (День 2-3)

### 2.1 Создание Context системы
**Файлы:** `contexts/*`
- ScreenSavyContext - глобальное состояние UI
- ThemeContext - тема и цвета
- Оптимизация с useReducer где нужно

### 2.2 Переиспользуемые хуки
**Файлы:** `hooks/*`
- useLocalStorage - типизированная работа с localStorage
- useFullscreen - управление fullscreen
- useColorState - управление цветовым состоянием

### 2.3 Разделение на компоненты
**Файлы:** `components/screensavy/ui/*`
- Clock → отдельный компонент

- Все панели → отдельные файлы
- Toolbar, Menu → отдельные файлы

### 2.4 Режимы работы
**Файлы:** `components/screensavy/modes/*`
- Каждый режим в отдельном файле
- Четкая изоляция логики
