# ScreenSavy.com

[English](README.md) | **Русский**

**Живой демо:** [https://www.screensavy.com/](https://www.screensavy.com/)

ScreenSavy.com — это веб-приложение для превращения любого дисплея в ambient-фон. Проект возрождает ностальгию по классическим скринсейверам и сочетает её с современными элементами управления, позволяя создавать цветовые заливки, анимированные переходы или функциональные оверлеи вроде часов и текста на ноутбуках, телевизорах, проекторах или планшетах.

Проект активно разрабатывается на русском и английском языках и фокусируется на легковесных, оффлайн-дружественных визуализациях, которые работают прямо в браузере. ScreenSavy идеален для создания атмосферного освещения, фотосессий, домашнего кинотеатра и творческих пространств, где настроенный фон поддерживает момент.

## Основные режимы

- **Один цвет** – установите один RGB-тон на весь экран. Идеально для калибровки монитора, ambient-освещения или цветотерапии.
- **Смена цвета** – создайте палитру любимых оттенков и позвольте ScreenSavy циклически переключаться между ними с настраиваемой скоростью.
- **Часы** – отображайте современные, полные или минималистичные показания времени с автоматическим форматированием даты и дня недели в зависимости от локали.
- **Текст** – создавайте типографические макеты с настраиваемыми шрифтами (28 вариантов), размерами, выравниванием, цветами и стилями для вывесок или экранов мероприятий.
- **Избранное и палитры** – сохраняйте образцы для повторного использования, исследуйте тональные вариации с панелью оттенков и точно настраивайте каналы с помощью числовых слайдеров.
- **Визуализаторы** – интерактивные аудио и ambient визуализации:
  - **Аудио-визуализаторы:** Celestial Weaver, Super Nova, Voyager
  - **Ambient-визуализаторы:** Lava Lamp, RGB Lava
- **Эргономика интерфейса** – переключайтесь в полноэкранный режим, скрывайте элементы управления для чистой презентации и полагайтесь на контекстные подсказки, которые направляют новых посетителей.

## Технологический стек

- **Фреймворк** – Next.js 14 с React 18 для гибридного статического/интерактивного опыта.
- **Стилизация** – Tailwind CSS наряду с ручными утилитарными классами, настроенными для полноэкранных макетов.
- **Состояние и хранилище** – клиентское состояние React с сохранением в localStorage для избранного, настроек UI и параметров анимации.
- **Интернационализация** – встроенные русские и английские переводы с автоматическим определением языка браузера.

## Портативная сборка для Windows

В репозитории появились два вспомогательных скрипта — `start.bat` и `update.bat`. С их помощью можно собрать архив с приложением и портативным Node.js, который запускается на Windows без установки дополнительных программ. Скрипты лежат в корне проекта и ожидают такую структуру внутри архива:

```text
ScreenSavy.com-portable/
├─ node/          (портативный Node.js из официального ZIP)
├─ .next/         (готовая production-сборка)
├─ node_modules/  (зависимости после `npm install --omit=dev`)
├─ start.bat      (запускает локальный сервер Next.js)
└─ update.bat     (скачивает или подтягивает свежую версию проекта)
```

* `start.bat` ищет рантайм в папке `node/`, при первом запуске устанавливает зависимости и собирает проект, а затем открывает сайт на `http://127.0.0.1:3000` и автоматически запускает браузер.
* `update.bat` умеет работать и в клоне репозитория, и в чистой портативной папке: при наличии `.git` выполняет `git pull`, иначе скачивает актуальный ZIP с GitHub, обновляет зависимости, удаляет устаревшую папку `.next` и собирает проект заново.

Полный чеклист для подготовки релизного архива описан в файле [`docs/PORTABLE_RELEASE.md`](docs/PORTABLE_RELEASE.md).

## Начало работы

### Требования

- Node.js 18.17 или новее (требование Next.js 14)
- npm 9 или новее

### Установка

Установите зависимости после того, как Node.js и npm станут доступны:

```bash
npm install
```

> **Примечание:** Среда разработки, использованная для создания этого проекта, не имела доступа к внешнему npm-реестру. Если у вас возникнут проблемы с установкой, убедитесь, что реестр доступен из вашей сети.

### Разработка

Запустите сервер разработки с горячей перезагрузкой:

```bash
npm run dev
```

По умолчанию приложение обслуживается на [http://localhost:3000](http://localhost:3000). Перейдите на `/` для основного цветового опыта или `/modes/text` для интерфейса текстового режима.

### Production-сборка

Создайте оптимизированную production-сборку и запустите её локально:

```bash
npm run build
npm start
```

Команда `npm start` запускает `next start`, который обслуживает предварительно собранное приложение.

### Линтинг

Запустите ESLint, используя конфигурацию Next.js:

```bash
npm run lint
```

Устраните все обнаруженные проблемы перед коммитом изменений.

1. Форкните репозиторий и создайте feature-ветку от `main`.
2. Установите зависимости и убедитесь, что `npm run lint` проходит локально.

4. Убедитесь, что коммиты используют осмысленные сообщения и ссылаются на связанные issue, когда они доступны.
